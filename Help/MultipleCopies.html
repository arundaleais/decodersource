Multiple Copies of AisDecoder

If multiple copies of AisDecoder are running on the same device (PC)

Input Restrictions

The Network input port must be unique for each copy. For an exception see TCP below.

You cannot use the same input port UDP for input on the same PC even if it is using a broadcast IP address (eg 192.168.1.255)

You can also run AisDecoder on the same PC if using TCP for input from the same server using the same port provided the server allows multiple connections to the same IP address. This depends on the configuration on the server. This is permissible because the specified port is only used to make the initial connection following which the sever and client (your PC), negotiate a mutuallly agreed free port. Suppliers of AIS data over the internet are unlikely to allow multiple connections (at no charge) due to server loading issues.

If multiple copies of AisDecoder are running on different devices on the same local network:-

You can use the same port for UDP if is using the broadcast address (eg 192.168.1.255) and AisDecoder is running on a different PC on the same local network.

File Restrictions

I have introduced a versioning naming convention to try and resolve output file name conflicts.

If AisDecoder detects a file is already in use it will add a version number by appending ;nnn to the file name immediately prior to the first extension name dot.

eg if the first file name is nmea.log the second will be nmea;1.log

This will only occur if the program cannot open the second file due to it being already in use.

If Cache Vessel Names is ticked on the Options window, AisDecoder will update the disk copy of these names from memory both hourly and when AisDecoder is terminated. When AisDecoder is started, AisDecoder reads the disk file into memory. Any new names received while AisDecoder is running will only be updated in memory.

This can result in vessel names received by one of two copies of AisDecoder running simultaneously not being updated on disk.

See also Options - UdpBroadcast
